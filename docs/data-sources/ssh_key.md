# nayatel_ssh_key (Data Source)

Look up an existing SSH key by name. Use this to reference a key registered outside Terraform; keys managed in the same configuration should use the `nayatel_ssh_key` resource instead.

## Example Usage

```terraform
# Reference an SSH key registered outside Terraform (e.g. via the portal).
data "nayatel_ssh_key" "personal" {
  name = "personal"
}

resource "nayatel_instance" "web" {
  name            = "web-server"
  image_id        = data.nayatel_image.ubuntu.id
  cpu             = 2
  ram             = 2
  disk            = 20
  network_id      = nayatel_network.main.id
  ssh_fingerprint = data.nayatel_ssh_key.personal.fingerprint
}
```
