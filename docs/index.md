# nayatel Provider

The Nayatel provider is a community-maintained, unofficial provider for interacting with Nayatel Cloud resources.

-> This is a community-maintained provider and is not affiliated with or supported by Nayatel. Resources created through it are billed to your Nayatel Cloud account.

!> This provider has only been developed and tested against a **Pay-As-You-Go (PAYG)** Nayatel Cloud account, where resources are billed as they're created (the provider previews cost and verifies balance before every create). Nayatel also offers subscription/monthly billing plans that provision and bill resources for a full month upfront; behavior on those plans (cost previews, balance checks, and billing amounts in particular) has not been verified and may differ.

## Example Usage

```terraform
terraform {
  required_providers {
    nayatel = {
      source = "matifali/nayatel"
    }
  }
}

# Credentials can be set here or via the NAYATEL_USERNAME and
# NAYATEL_PASSWORD environment variables.
provider "nayatel" {
  username = "your-username"
  password = "your-password"
}
```

## Authentication

The provider needs a `username` and `password`. Credentials can be set in the provider block or via environment variables:

```shell
export NAYATEL_USERNAME="your-username"
export NAYATEL_PASSWORD="your-password"
```

-> The session token obtained at login is cached under `~/.config/nayatel/` and reused until it expires, so repeated Terraform runs do not re-authenticate.
